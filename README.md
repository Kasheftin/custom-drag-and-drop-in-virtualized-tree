# Drag & Drop + LexoRank Laboratory

A proof of concept for data-driven drag-and-drop and rank maintenance in a virtualized, collapsible tree table. It is built with Vue 3 and TypeScript without a drag-and-drop library.

**[Original live demo](https://kasheftin.github.io/custom-drag-and-drop-in-virtualized-tree/)**

## What it demonstrates

- **Virtual scrolling** — only visible rows are in the DOM; the tree handles hundreds of thousands of items
- **Collapsible tree** — depth-first flat rendering with animated expand/collapse
- **Drag ghost** — a custom ghost follows the mouse and previews a bounded number of rows
- **Drop highlight** — a geometry-driven overlay uses `document.elementsFromPoint` without changing table layout
- **Depth-aware drop** — horizontal movement selects the intended hierarchy depth
- **Reference LexoRank ordering** — complete bucket-prefixed ranks are generated by [`kvandake/lexorank-ts`](https://github.com/kvandake/lexorank-ts)
- **Independent sibling domains** — root items and every parent's children have unrelated rank spaces and bucket lifecycles
- **Inline rank editing** — click any rank to create a deliberate duplicate, dense interval, or bucket mismatch
- **Densify helper** — inject a run of long, adjacent ranks into one interval to reproduce a hot spot without hundreds of manual drags
- **Partial rerank** — density- and collision-driven local repair: walks the whole dense/duplicate run, redistributes it between healthy anchors, and escalates to a full rerank past a maximum window
- **Full rerank of any domain** — migrate the root list or any parent's children through the `0 → 1 → 2 → 0` bucket cycle in UI-friendly batches
- **Sparse context state** — missing state means bucket 0; explicit state exists only during an operation or while a nonzero bucket is established

## Tech

- Vue 3 `<script setup>` + TypeScript
- Vite
- `lexorank` 1.0.5
- Vitest
- No external drag-and-drop library
- Custom `useDrag`, `useEdgeScroll`, and `useDropHighlight` composables
